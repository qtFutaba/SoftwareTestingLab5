# SoftwareTestingLab5
Lab 5 for Software Testing class. Use Postman to test input/output of API system.

INCLUDES: 
-Code repository to get it working (You may notice: it's just Dr. Baarsch's code!)
-Test Case Document (Caiden Clement - Software Testing Lab 5)
-Postman Collection JSON Files (Student, Course, API Functionality)

TO USE: 
- Open the code in whatever IDE you have, whatever way you want. Copy and paste it in, clone the repo, I don't care.
- Run the "StudentRegDemo" file.
- Open Postman.
- Import the test collections.
- Run them in order from top to bottom. If you read the test case document, you will note that the order is IMPERATIVE!
- NOTE: There are some test cases that require two runs of the same test following some condition. Read the document!

This suite should test all major functionality and constraints of the code.
